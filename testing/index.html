Testing in the browser with Karma and Jasmine

Tools

  • Karma - tool used to spawn a web server which loads your application's source code and executes your tests.
  • Jasmine - BDD framework for JavaScript, provides functions to help with structuring tests and making assertions.

Structuring tests


describe('A suite', () => {

      it('contains spec with an expectation', () => {
        expect(true).toBe(true);
      });

});

Karma configuration


// install Karma
npm install karma
		  

// install plugins
npm install karma-jasmine
npm install karma-phantomjs-launcher
npm install karma-chrome-launcher
		  

// run Karma
./node_modules/karma/bin/karma start / init / run
// alternative
npm install -g karma-cli
karma start / init / run

Karma configuration file


// can be created with karma init command
module.exports = function (config) {
	config.set({
		basePath: '',
		frameworks: [],
		files: [],
	    hostname: 'localhost',
		port: 9876,
		autoWatch: false,
		browsers: [],
		singleRun: false,
	})
};
// for debugging in a browser:
// - set single run to true
// - select other browser

Testing on node with Mocha and Chai

Tools

  • Mocha - is a feature-rich JavaScript test framework running on Node.js and in the browser.
  • Chai - is a BDD / TDD assertion library for node and the browser that can be delightfully paired with any javascript testing framework.

Configuration


// install Mocha and Chai
npm install mocha
npm install chai
		  

// run tests
./node_modules/mocha/bin/mocha //runs all test from default 'test' dir
// alternative
npm install -g mocha
mocha tests --recursive --watch 
// runs all tests from 'tests' directory
// + subdirectories and watches files for changes

Structuring tests


const chai = require('chai');

const expect = chai.expect; 
const should = chai.should; 
const assert = chai.assert; 

describe('A suite', () => {

      it('contains spec with an expectation', () => {
        expect(foo).to.be.a('string');
        foo.should.be.a('string');
        assert.equal(foo, 'bar');
      });

});

Exlusions / inclusions


describe.only('A suite', () => {
    // only this suite will be run
});

describe.skip('A suite', () => {
    // the suite will be skipped
});

it.only('spec', () => {
    // only this spec will be run
});

it.skip('spec', () => {
    // this spec will be skipped
});

Debbuging with VS Code

  1. Set some breakpoints in VS editor
  2. Swich to the debug view
  3. Expand configuration dropdown and select 'Add Configuration'
  4. Start typing mocha - example config is automatically generated
  5. Run the config
